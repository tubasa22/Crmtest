## ⚡ 이번주 할 일 (30분)

### 1️⃣ 브랜치 설정 (5분)
```bash
git checkout -b develop
git push -u origin develop
```
**결과:** main + develop 2개 브랜치

### 2️⃣ 수정 파일 올리기 (10분)
```bash
# index_fixed.html → index.html로 사용
cp index_fixed.html index.html

git add index.html
git commit -m "fix: 생일/PAK ANNE 오류 수정"
git push origin develop
```
**결과:** develop 브랜치에 최신 코드 저장

### 3️⃣ PR로 배포 (10분)
```
GitHub 웹사이트:
1. Pull requests → New pull request
2. develop → main 선택
3. Create pull request
4. Merge pull request
```
**결과:** 수정 내용이 main에 반영되어 배포됨

### 4️⃣ code.gs 백업 (5분)
```bash
# Google Apps Script 에디터에서 코드 복사
# → apps/code.gs.backup 파일에 붙여넣기

mkdir -p apps
# 편집기에서 코드 복사 후:
nano apps/code.gs.backup
# 붙여넣기 → Ctrl+X, Y, Enter

git add apps/code.gs.backup
git commit -m "chore: Google Apps Script 백업"
git push origin develop
```
**결과:** 앱스 스크립트 코드 백업 완료

## 🔄 깃 브랜치 전략

### 현재 구조 (❌ 좋지 않음)
```
main (프로덕션)
```

### 개선된 구조 (✅ 권장)
```
main (프로덕션 - 배포된 코드만)
  ↑
develop (개발 버전 - 다음 배포 예정)
  ↑
feature/[기능명] (새 기능 개발)
```

### 사용 방법
```bash
# 새 기능 개발 시
git checkout -b feature/새기능명
# ... 개발 ...
git commit -m "feat: 새 기능"
git push origin feature/새기능명
# GitHub에서 PR 생성 → develop에 merge

# 버그 수정 시
git checkout -b bugfix/버그설명
# ... 수정 ...
git commit -m "fix: 버그 설명"
git push origin bugfix/버그설명
# GitHub에서 PR 생성 → develop에 merge

# 배포할 때 (develop → main)
# PR 생성 후 merge
```
